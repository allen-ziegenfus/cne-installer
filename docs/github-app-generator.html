Setup Instructions

  1. Register: Click "Register GitHub App" above. This will take you to GitHub to create the app.
  2. Install: Once created, navigate to "Install App" in your new app's settings and install it on your organization.
  3. Finalize: After installation, you will be redirected to a success page containing a code. Use this code in the following curl command to retrieve your App ID and Private Key: 
    curl -X POST \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/app-manifests/YOUR_CODE_FROM_URL/conversions

Why these permissions?

Contents (read): Allows ArgoCD to read your Git repository to sync Helm charts and Kubernetes manifests.
Metadata (read): Basic requirement for all GitHub Apps to identify the repository and its basic structure.
Members (read): Required for ArgoCD **SSO (Single Sign-On)**. It allows ArgoCD to verify which GitHub Teams a user belongs to for RBAC (Role-Based Access Control).
Emails (read): Required for SSO to map your GitHub account to your ArgoCD user profile using your primary email.
Actions & Variables (write): Used by the automated installer to configure GitHub Actions secrets and variables needed for the CI/CD pipeline (e.g., pushing images to Artifact Registry).
